# github_DanielN6767_vjezba_1
Upoznavanje osnova githuba za skolu.
1.Što je Vsual Studio Code?
Visual Studio Code je otvreni i besplatni uređivač izvornog koda.
2.Kako se zove panel u kojem vidiš sve datoteke i mape svog projekta?
Panel se zove Explorer.
3.Kako možeš instalirati ekstenzije u VS Codeu i čemu one služe?
Ekstenzije instaliraš tako da:klikneš na ikonu Extensions (kvadratić sa 4 kockice),upišeš naziv ekstenzijeklikneš Install.
Služe za dodatne funkcije poput podrške za jezike, teme, alate, Git poboljšanja itd.
4.Kako otvoriti postojeći projekt ili mapu u VS code-u?
Povuci mapu u VS Code prozor.
5.što se sve može raditi u Exploreru
U Exploreru možeš:pregledavati datoteke i mape,stvarati nove datoteke/mape,brisati, premještati i preimenovati datoteke,otvarati datoteke za uređivanje.
6.Kako možeš brzo pronaći određenu datoteku u projektu?
Pritiskom na Ctrl + P (Quick Open), zatim upišeš ime datoteke.
7.Kako možeš istovremeno gledati dvije različitr datoteke u projektu?
Povuci karticu datoteke u desnu stranu prozora.
8.kako pokrećeš automatsko formatiranje koda u VS code-u?
Shift + Alt + F
9.Što je version control i zašto se koristi?
Version control (kontrola verzija) je sustav za praćenje promjena u kodu.
Koristi se kako bi:vidio tko je što promijenio,mogao vratiti staru verziju,mogao raditi u timu bez konflikata.
10.Kako se u VS code-u otvara panel za Git(Source control)?
Klikom na Source Control ikonu (ikona grananja) na lijevom panelu ili tipkom Ctrl + Shift + G.
11.Što znače commit, push , pull?
Commit – spremanje promjena lokalno u Git povijest.

Push – slanje lokalnih promjena na udaljeni repozitorij (GitHub).

Pull – preuzimanje najnovijih promjena s GitHuba na računalo.
12.Čemu služi commit poruka i kako treba izgledati?
Služi za opis onoga što si promijenio.
Treba biti kratka, jasna i konkretna.
Primjer:

"Dodana funkcija za izračun prosjeka"
13.Kako možeš vidjeti koje si promjene napravio u pojedinoj datoteci?
U Source Control panelu klikneš na datoteku, a VS Code prikazuje dif (razlike).
14.Kreiraš novu datoteku. Koji su koraci da se promjena pošalje na GitHub?
Spremi datoteku (Ctrl + S)

Otvori Source Control

Dodaj promjene (Stage changes)

Napiši commit poruku

Klikni Commit

Klikni Push
15.Kako se vraćaš na stariju verziju datoteke ako si pogriješio?
U Source Controlu otvori povijest (View File History / Timelines)

Odaberi stariju verziju

Klikni Restore ili kopiraj sadržaj te verzije.
16.Na koji način možeš provjeriti status projekta i koje datoteke su izmijenjene?
U Source Control panelu vidiš sve izmijenjene datoteke.
Alternativno: otvori terminal i upiši git status.
